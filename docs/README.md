## 🎯 Design Principles

1. **Documentation drives implementation** — Write docs first, code second.
2. **Be thorough, not clever** — Clear and comprehensive beats elegant and terse.
3. **Don't skip things** — If it's important, document it.
4. **Cross-reference liberally** — Link to other docs when relevant.
5. **Keep it current** — Update docs as the project evolves.

---

## 📝 How to Update These Docs

- **When adding a feature:** Update PRODUCT.md, CONTENT_MODEL.md (if data changes), IMPLEMENTATION.md
- **When making architectural changes:** Update ARCHITECTURE.md, DECISIONS.md
- **When changing tech:** Update TECH_STACK.md, DECISIONS.md
- **When establishing patterns:** Update CONVENTIONS.md
- **When completing tasks:** Update IMPLEMENTATION.md

Every significant change should result in updated documentation.
